# README
## aerial_counts_data
En esta tabla se enseña, por día de censo aéreo (columna: “Fecha”): las especies (columna: “Especie”) y el número de individuos detectados por cada una de ella (columna: “Individuos”) en cada localidad (polígono) en los 3 niveles de anidamiento (columna: “Localidad nivel 1”, “Localidad nivel 2”, “Localidad nivel 3”), el ambiente dominante en la localidad de nivel 3 (columna: “Ambiente”); el código de censo (columna: Código de censo); el mes (columna; “Mes”) y el año (columna: “Año”) y si el censo es válido (TRUE) o no valido (FALSE) (columna: “Censo valido”).
La tabla recopila solamente datos de presencia. En caso de necesitar un base completa con datos de presencia, ausencia y de no disponibilidad de dato (NA) por cada especie de aves, habria que cruzar la información de esta tabla con la que aparece en el fichero de descarga “aerial_counts_localities” donde se especifica por cada día de censo si la localidad de nivel 3 ha sido censada o no.
Cada censo se identifica con un código alfanumérico y se compone de las iniciales de censo aéreo (“CA”) seguidas por el año, el mes y el día de censo.

## aerial_counts_localties
En esta tabla se enseña, por cada censo (columna: “Código de censo”), si la localidad de nivel 3 ha sido censada (“VERDADERO”) o no (“FALSO”) (columna: “Se censa”); si había (“SI”) o no había agua (“No”) en la localidad de nivel 3 (columna: “agua”) y el ambiente dominante (columna: “Ambiente”). 
Cada censo se identifica con un código alfanumérico y se compone de las iniciales de censo aéreo (“CA”) seguidas por el año, el mes y el día de censo.

## aerial_counts_metadata
En esta tabla se presenta toda la información adicional disponible sobre cada censo. Por cada censo (columna: “Código de censo”) se indica: la fecha del censo (columna: “Fecha”), el censador (columna: “Censador”), los acompañantes (columnas: “Acompañante 1” y “Acompañante 2”), el piloto (columna: “Piloto”), el modelo de avioneta (columna: “Avioneta”); la matricula del avioneta (columna: “Matricula”), el Aeropuerto de salida y vuelta (columna: “Aeropuerto”), la hora de inicio y fin del censo (columnas: “Hora inicio” y “Hora fin”), si está disponible el recorrido del vuelo (“TRUE”) o no (“FALSE”) (columna: “Track GPS”), el nombre del recorrido (columna: “Track”), si está disponible la foto del censo “TRUE” o no “FALSE” (columna: Foto), las condiciones de visibilidad (columna: “Visibilidad”), la intensidad del viento  (columna: “Intensidad del viento”), la dirección del viento (columna: “direccion del viento”, la condición atmosférica de ese día (columna: “condición atmosférica”), la temperatura (columna: “Temperatura”), si el censo es válido (“TRUE”) o no (“FALSE”) (columna: “Censo valido”) y por ultimo unas observaciones sobre el censo (columna: “Obervaciones”). 
Cada censo se identifica con un código alfanumérico y se compone de las iniciales de censo aéreo (“CA”) seguidas por el año, el mes y el día de censo.
